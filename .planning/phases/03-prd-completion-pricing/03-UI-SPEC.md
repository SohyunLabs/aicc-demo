## Spacing Scale (Document Hierarchy)

Markdown documents use whitespace and heading levels instead of pixel spacing. The following contract ensures consistent visual rhythm.

| Token | Markdown Equivalent | Usage |
|-------|-------------------|-------|
| Section break | 2 blank lines before `## H2` | Major topic transitions (e.g., "자산 활용 계획" to "우선순위 매트릭스") |
| Subsection break | 1 blank line before `### H3` | Subtopic transitions (e.g., "보이스봇 기능" to "Agent Assist 기능") |
| Paragraph gap | 1 blank line between paragraphs | Standard prose separation |
| List item gap | No blank lines between items | Tight lists for scan-readability |
| Table buffer | 1 blank line before and after table | Visual separation of structured data |
| Blockquote buffer | 1 blank line before and after `>` block | Citation/data point isolation |

Heading levels (exactly 3, matching Phase 1/2):

| Level | Usage | Example |
|-------|-------|---------|
| `# H1` | Document title only (1 per file) | `# 뤼튼 AICC 가격 전략` |
| `## H2` | Major sections (numbered: 1, 2, 3...) | `## 1. 기존 자산 활용 계획 (PRD-05)` |
| `### H3` | Subsections (numbered: 1.1, 1.2...) | `### 1.1 Build/Enhance 분류 매트릭스` |
| `#### H4` | Rare: sub-subsection or table title | `#### 시장 규모 수치 비교표` |

Exceptions: none

## Typography (Document Formatting)

Markdown rendering is font-agnostic. This contract defines emphasis and formatting patterns instead.

| Role | Formatting | Usage |
|------|-----------|-------|
| Body text | Plain paragraph | Korean prose, 1-3 sentences per paragraph for scannability |
| Key metric | `**bold**` wrapping | Numbers, percentages, currency: `**3조5,349억원**`, `**69%**`, `**₩1,500/건**` |
| Technical term | Plain (first use) or backtick for code-like terms | `Hybrid RAG`, `STT/TTS`, `MKT-DP-01` |
| Emphasis | `**bold**` | Strategic conclusions, product names at first mention |
| Data point label | `> **[TAG-XX]**` blockquote format | `> **[MKT-DP-01]** 한국 컨택센터...` |
| Table header | Pipe table with alignment | Left-align text, left-align numbers (Korean convention) |

**Language rules (from CLAUDE.md):**
- Body text: Korean
- Technical terms, variable names, product names: English where standard (e.g., "Hybrid RAG", "FCR", "AHT")
- Currency: `₩` prefix for Korean Won, `$` for USD
- Large Korean Won amounts: `만원` or `억원` units (e.g., `₩3,000만원`, `5억원`)

## Color (Document Formatting)

Markdown documents have no color control. Visual hierarchy is achieved through formatting weight.

| Role | Formatting Equivalent | Usage |
|------|----------------------|-------|
| Dominant (60%) | Plain body text | Narrative prose, explanations |
| Secondary (30%) | Tables + blockquote citations | Structured data, evidence citations |
| Accent (10%) | **Bold key numbers** + bullet highlights | Critical metrics, strategic conclusions |
| Destructive | Not applicable | No destructive actions in document phase |

Accent (bold) reserved for:
- Market size numbers (e.g., `**3조5,349억원**`)
- KPI target numbers (e.g., `**78%**`)
- Price points (e.g., `**₩1,800/건**`)
- Percentage changes (e.g., `**+12.05%**`)
- Strategic conclusion phrases (e.g., `**빈 공간이 뤼튼 AICC의 전략적 기회다**`)

Bold must NOT be used for: generic emphasis, section headers (headers self-emphasize), table cell contents (table structure provides emphasis).

### Citation Format (MKT-DP continuation)

Phase 1 established `MKT-DP-XX` (Market Data Points). Phase 3 introduces two new tag families:

| Tag Family | Format | Usage | Example |
|-----------|--------|-------|---------|
| KPI 벤치마크 | `[KPI-BP-XX]` | KPI benchmark data points | `> **[KPI-BP-01]** 금융 산업 FCR 산업 평균: **71%**` |
| 가격 벤치마크 | `[PRC-BP-XX]` | Pricing benchmark data points | `> **[PRC-BP-01]** Intercom Fin AI 해결건당: **$0.99/건**` |
| 기존 MKT 참조 | `(MKT-DP-XX 참조)` | Cross-reference to Phase 1 data | `KT가 69%를 점유하고 있으나(MKT-DP-04 참조)` |

Citation blockquote format (exact pattern, continuing Phase 1):

```markdown
> **[KPI-BP-01]** 금융 산업 FCR(First Call Resolution) 산업 평균: **71%**
> -- 출처: SQM Group, Call Center FCR Benchmark 2024 Results by Industry (HIGH)
```
