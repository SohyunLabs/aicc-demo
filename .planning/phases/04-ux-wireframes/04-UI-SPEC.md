# Phase 4 — UI Design Contract

> Visual and interaction contract for UX wireframe document production. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 4 deliverable context:** 이 Phase의 산출물은 실제 코드 UI가 아니라 고밀도 ASCII 와이어프레임이 포함된 마크다운 기획 문서다. 본 UI-SPEC은 (1) 와이어프레임 내부의 정보 구조/레이아웃/카피 규약과 (2) 향후 구현 시 참조할 디자인 시스템 매핑을 모두 정의한다.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 아이콘과 인라인 텍스트 간격, 태스크 번호 마커(1)(2)(3) 내부 패딩 |
| sm | 8px | 와이어프레임 내 밀접 요소 간격 (라벨-값 쌍, 버튼 그룹 내부) |
| md | 16px | 기본 카드 내부 패딩, 폼 필드 간 간격, 와이어프레임 박스 내부 여백 |
| lg | 24px | 섹션 패딩, 사이드바/메인 영역 내부 여백 |
| xl | 32px | GNB와 콘텐츠 영역 간 갭, 온보딩 스텝 간 구분 |
| 2xl | 48px | 화면 간 주요 구분선, 와이어프레임 문서 내 섹션 브레이크 |
| 3xl | 64px | 페이지 최상단/최하단 여백 |

Exceptions:
- 터치 타겟(모바일 고려 시): 44px 최소 높이 -- 단, Phase 4는 데스크톱 웹 기반 관리 콘솔 우선이므로 와이어프레임에서는 적용하지 않음
- Agent Assist 컴플라이언스 경고 배너: 최소 높이 40px -- 상담원 즉시 인지를 위한 예외

## Typography

| Role | Size | Weight | Line Height | 와이어프레임 표현 |
|------|------|--------|-------------|-----------------|
| Body | 14px | 400 (regular) | 1.5 | 일반 텍스트, 테이블 셀 내용, 설명문 |
| Label | 12px | 600 (semibold) | 1.4 | 필드 라벨, 상태 뱃지, KPI 단위, 사이드바 메뉴 항목 |
| Heading | 20px | 600 (semibold) | 1.2 | 섹션 제목, 카드 헤더, 패널 타이틀 |
| Display | 28px | 600 (semibold) | 1.2 | KPI 숫자(해결률 65%, AHT -30%), 페이지 타이틀 |

**와이어프레임 내 타이포그래피 표현 규칙:**
- Display: `[== KPI 숫자 ==]` 또는 ASCII 박스 내 굵은 경계선으로 표현
- Heading: 언더라인(`---`) 또는 `[ 섹션 제목 ]` 형태
- Body: 일반 텍스트
- Label: 대문자 또는 `[LABEL]` 형태로 구분

- GNB 활성 메뉴 인디케이터

- Accent/Primary: `[*강조 요소*]` 또는 `>>>` 화살표로 시선 유도
- Destructive: `[!경고!]` 또는 `[!! 컴플라이언스 위반 !!]` 형태
- Secondary: 박스 테두리(`+---+`)로 카드/패널 영역 표현
- Muted: `(비활성 텍스트)` 형태

```
| (4) 지식 관리                |                               |
| (5) 설정                     |                               |

```

**GNB 5대 영역 (CONTEXT.md 결정사항):**

| 메뉴 | 아이콘 (lucide) | 설명 |
|------|----------------|------|
| 대시보드 | LayoutDashboard | KPI 요약, 최근 활동, 시스템 상태 |

- 노드 유형: 시작(원형) / 의도 분류(마름모) / 응답(사각형) / 상담원 전환(육각형) / 종료(이중 원)
- 노드 내부: 노드명(Heading) + 조건/응답 요약(Body) + 편집 아이콘
- 엣지: 조건 라벨 표시 (예: "고객 의도: 해지 문의")
- 캔버스 도구바: 노드 추가, 확대/축소, 정렬, 실행 취소, 배포

### UX-02: Agent Assist 실시간 보조 화면

**3칸럼 레이아웃 (CONTEXT.md 결정사항):**
```
+--좌측 패널 (30%)--+--중앙 패널 (40%)--+--우측 패널 (30%)--+
| [보이스봇 대화 요약]|                    |                    |
| ================== |                    |                    |
| [실시간 전사]       | [추천 응답]         | [고객 정보]         |
|  - 고객 발화       |  - RAG 기반 응답    |  - 이름/등급/이력   |
|  - 상담원 발화     |  - 출처 문서 링크   |  - 최근 거래       |
|                    | ================== |                    |
|                    | [컴플라이언스 경고]  | [감정 분석]         |
| [실시간 요약]      |  - 인라인 알림      |  - 감정 게이지      |
|  - 핵심 키워드     |  - 규정 조항 안내   |  - 추이 그래프      |
+--------------------+--------------------+--------------------+
| [통화 상태 바: 통화 시간 | 녹음 중 | 보이스봇→상담원 전환 이력] |
+----------------------------------------------------------------+
```

**상담원 시선 흐름 (좌->우) 기반 정보 배치:**
1. 좌측: 지금 무슨 대화가 오가는지 (전사 + 요약) -- 가장 먼저 봄
2. 중앙: 뭘 해야 하는지 (추천 응답 + 경고) -- 행동 유도
3. 우측: 참고 맥락 (고객 정보 + 감정) -- 필요 시 참조

**컴플라이언스 경고 인라인 표현 (CONTEXT.md 결정사항):**
- 전사 텍스트 내: 위반 발화에 빨간 하이라이트 마커 `[!! 설명의무 누락 !!]`
- 중앙 패널 상단: 경고 배너 `[! 보험업법 제95조 참조: 면책조항 설명 필요 !]`
- 경고 해제 조건: 상담원이 해당 내용을 설명하면 자동 해제 + 체크 표시

**보이스봇 맥락 인수 카드 (CONTEXT.md 결정사항):**
- 위치: 좌측 패널 최상단 (전사 영역 위)
- 내용: 보이스봇 대화 요약 / 고객 의도 / 질문 내용 / 보이스봇 응답 요약
- 표현: 배경색 구분 (muted) + "보이스봇에서 전환됨" 뱃지

## Wireframe Annotation Convention

Phase 4 와이어프레임은 ASCII 텍스트 기반이므로, 다음 주석 규약을 적용한다.

| 표기법 | 의미 | 예시 |
|--------|------|------|
| (1)(2)(3) | 태스크 번호 마커 | (1) 시나리오 노드를 드래그하여 추가 |
| `[버튼 라벨]` | 클릭 가능한 버튼/링크 | `[시나리오 배포]` |
| `[*강조 버튼*]` | Primary CTA (accent 색상) | `[*테스트 콜 시작*]` |
| `[!경고!]` | Destructive/경고 요소 | `[!면책조항 설명 필요!]` |
| `>>>텍스트<<<` | 시선 유도/추천 강조 | `>>>추천<<<` |
| `(설명 텍스트)` | 비활성/보조 텍스트 (muted) | `(아직 데이터 없음)` |
| `+---+` | 카드/패널 경계선 | 박스 레이아웃 |
| `======` | 섹션 구분선 (카드 내) | 패널 내부 영역 구분 |
| `-->` | 플로우 방향/연결 | 노드 간 엣지, 스텝 진행 |
| `[v]` | 완료 상태 체크 | 스텝 위자드 완료 단계 |

## Interaction States

### 공통 상태 정의

| 상태 | 시각적 표현 | 와이어프레임 표기 |
|------|-----------|-----------------|
| Default | 기본 스타일 | 일반 박스/텍스트 |
| Hover | 배경색 변경 (secondary) | 와이어프레임에서 생략 |
| Active/Selected | Accent 테두리 + 배경 | `[*선택된 항목*]` |
| Disabled | Muted 텍스트 + 테두리 | `(비활성 항목)` |
| Loading | 스켈레톤 또는 스피너 | `[...로딩 중...]` |
| Error | Destructive 테두리 + 메시지 | `[!에러 메시지!]` |
| Empty | 중앙 정렬 안내 텍스트 + CTA | 별도 empty state 박스 |

### 화면별 핵심 인터랙션

**UX-01 시나리오 빌더:**
1. 노드 드래그 앤 드롭 -> 캔버스에 추가
2. 노드 클릭 -> 우측 속성 패널 열림
3. 엣지 드래그 -> 노드 간 연결
4. 배포 버튼 -> 확인 모달 -> 배포 완료 토스트

**UX-02 Agent Assist:**
1. 통화 연결 -> 전사 실시간 스트리밍 시작
2. RAG 검색 완료 -> 추천 응답 카드 페이드인
3. 컴플라이언스 위반 감지 -> 인라인 경고 즉시 표시 (최우선)
4. 추천 응답 "사용" 클릭 -> 응답 텍스트 클립보드 복사 + 확인 표시

**UX-03 온보딩:**
1. 스텝 완료 -> 인디케이터 체크 + 다음 스텝 활성화
2. 산업 카드 클릭 -> 선택 하이라이트 + 포함 기능 목록 확장
3. 테스트 콜 시작 -> 시뮬레이터 활성화 + AI 응답 실시간 표시
4. 결제 티어 선택 -> 가격 정보 강조 + "구독 시작" CTA 활성화
