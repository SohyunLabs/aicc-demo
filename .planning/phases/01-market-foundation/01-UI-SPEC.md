## Spacing Scale

Declared values (Markdown structural equivalents):

| Token | Markdown Equivalent | Usage |
|-------|---------------------|-------|
| xs | no blank line | Consecutive lines within a paragraph |
| sm | 1 blank line | Between paragraphs within a subsection |
| md | 1 blank line + `###` heading | Between subsections |
| lg | 2 blank lines + `##` heading | Between major sections |
| xl | 2 blank lines + `---` + `##` heading | Between top-level document parts |
| 2xl | not used | -- |
| 3xl | not used | -- |

Exceptions: ASCII positioning maps use manual spacing (spaces/pipes) for diagram alignment. No constraint on internal diagram whitespace.

## Typography

Markdown heading hierarchy for the strategic proposal:

| Role | Markdown | Usage | Constraint |
|------|----------|-------|------------|
| Document title | `# H1` | Document title only. Exactly 1 per document. | "한국 AICC 시장 분석 및 차별화 전략" |
| Major section | `## H2` | Top-level sections (Executive Summary, 시장 기회, 경쟁 구도, White Space, 전략적 시사점, 출처). Maximum 7. | Numbered prefix: `## 1. 시장 기회: ...` (except Executive Summary and 출처) |
| Subsection | `### H3` | Subsections within H2. Maximum 4 per H2. | Numbered prefix: `### 1.1 시장 규모 및 성장률` |
| Sub-subsection | `#### H4` | Rare. Only for deeply nested content (e.g., individual competitor within comparison). Maximum 3 per H3. | Use sparingly; prefer tables over nested headings |
| Body text | Plain paragraph | All analytical prose | Korean language. Sentences under 60 characters where possible for readability. |
| Data callout | `> **[MKT-DP-XX]**` blockquote | Numbered data points for cross-phase citation | Must include source footnote reference |
| Footnote | `[^N]` + footnote definition | Source citations | Sequential numbering, definitions at document end |

**Weights (emphasis):**
- `**bold**` -- reserved for: data point IDs, key figures (numbers + units), company names on first mention in a section
- `*italic*` -- reserved for: foreign terms, report titles, product names
- Do NOT combine bold and italic. Do NOT use bold for entire sentences.

**Line length:** Target 80 characters per line in source Markdown for diff readability. Hard wrap NOT required (soft wrap acceptable).

---

## Color

Not applicable -- Markdown documents have no color control. However, the following semantic conventions apply:

| Role | Markdown Convention | Usage |
|------|---------------------|-------|
| Emphasis (accent equivalent) | `**bold**` | Key figures, data point IDs |
| De-emphasis (muted equivalent) | `> blockquote` | Data point callouts, side notes |
| Warning/caution | inline note: `(주의: ...)` | Caveats about data reliability (MEDIUM/LOW confidence) |
| Structural separator | `---` horizontal rule | Between major document parts only. Maximum 3 per document. |

Accent reserved for: data point IDs (`[MKT-DP-XX]`), market size figures, growth rates, and competitor market share percentages. Never bold entire paragraphs.

## Copywriting Contract

### Document-Level Copy

| Element | Copy (Korean) |
|---------|---------------|
| Document title | 한국 AICC 시장 분석 및 차별화 전략: 성과 기반 과금 x 컴플라이언스 엔진의 기회 |
| Executive summary opening | Perplexity 비유로 시작 -- "범용 LLM이 답을 만들어낸다면, 도메인 특화 RAG는 출처와 함께 답을 찾아준다." 형태의 한 문장 hook |
| Executive summary structure | 3-bullet 핵심 메시지: (1) 시장 기회 크기, (2) 경쟁 구도 빈 공간, (3) 뤼튼의 포지셔닝 |
| Section transition pattern | 각 H2 섹션 첫 문단은 "이 섹션에서 다루는 내용"이 아닌 "핵심 인사이트 한 문장"으로 시작 |

### Data Point Formatting

| Element | Format |
|---------|--------|
| Data point callout | `> **[MKT-DP-XX]** {수치 포함 한 문장 팩트}` + 줄바꿈 + `> -- 출처: {출처명}, {연도}` |
| Inline data reference | `{서술}[^N]` -- 본문 흐름 내에서 각주 번호로 인용 |
| Cross-phase citation anchor | 각 데이터 포인트에 `MKT-DP-01` ~ `MKT-DP-18+` 고유 ID. 후속 Phase에서 `MKT-DP-01 참조` 형태로 인용 |
| Confidence indicator | 데이터 포인트 출처 옆에 신뢰도 표기: `(HIGH)`, `(MEDIUM)`, `(LOW)` |

### Table Formatting

| Element | Convention |
|---------|------------|
| Comparison table | 첫 열은 비교 축(구분), 나머지 열은 비교 대상. 열 수 최대 8 (가독성 한계). |
| Market sizing table | 연도별 행, 출처별 열. 단위 명시 필수 (억원, $M 등). |
| Alignment | 숫자 열은 우측 정렬 권장 (Markdown 지원 시). 텍스트 열은 좌측 정렬. |
| Empty cell | `-` (하이픈). 빈 칸으로 두지 않는다. |

### ASCII Diagram Contract

| Element | Convention |
|---------|------------|
| Positioning map | 2축(X: 과금 모델 혁신도, Y: 컴플라이언스 특화 수준). 경쟁사를 좌표에 배치. 뤼튼 목표 위치에 `★` 표기. |
| Map dimensions | 최소 15행 x 50열. 축 라벨 필수. |
| Market flow diagram | 사용 시 `→` 화살표 + 박스(`[ ]`) 형태. 3단계 이하. |

### Tone and Voice

| Dimension | Contract |
|-----------|----------|
| 어조 | 전략 제안서 -- 경영진/리더십 대상. 설득력 있되 과장 금지. |
| 주어 | 3인칭 서술 기본. "뤼튼은 ~할 수 있다" 형태. 1인칭("우리") 사용하지 않는다. |
| 수치 표현 | 원화: `X조 X,XXX억원` 또는 `X,XXX억원`. 달러: `$X.XM` 또는 `$X.XB`. 백분율: `XX.X%`. |
| 부정적 서술 | 경쟁사 약점은 "~에 한계가 있다", "~이 부재하다" 형태. 비하/공격적 표현 금지. |
| 인용 주의 | 자사 마케팅 자료 기반 수치(솔트룩스 99.9% 정확성 등)는 반드시 `(자사 발표, 독립 검증 미완)` 부기. |

### Section-Specific Copy Anchors

| Section | Opening Sentence Pattern | Closing Pattern |
|---------|-------------------------|-----------------|
| Executive Summary | Perplexity 비유 hook + 3-bullet 핵심 | "이하 본문에서 각 논점을 데이터와 함께 상세히 논증한다." |
| 시장 기회 | TAM 수치로 시작 ("한국 컨택센터 시장은 2025년 X조원 규모로...") | SAM/SOM 축소 논리로 마무리 |
| 경쟁 구도 | KT 69% 팩트로 시작 ("KT가 구축형 AICC B2B 시장의 69%를 점유한 가운데...") | "이러한 경쟁 구도에서 빈 공간은 어디인가?" 전환 질문 |
| White Space | 포지셔닝 맵 ASCII 도표 먼저 제시, 이후 텍스트로 해석 | Perplexity 비유 재등장 -- 포지셔닝 요약 |
| 전략적 시사점 | "Phase 1 분석의 핵심 시사점은 세 가지로 요약된다." | 후속 문서(PRD, 가격 전략) 연결점 명시 |
| 출처 | 각주 번호 순서대로 나열 | 신뢰도 등급(HIGH/MEDIUM/LOW) 범례 포함 |

## Document Structure Contract

이 섹션은 Phase 1 고유 -- 문서 산출물의 구조 계약.

### Required Sections (in order)

```
# 한국 AICC 시장 분석 및 차별화 전략

## Executive Summary                    (1-2 pages)
## 1. 시장 기회: 한국 AICC 시장의 폭발적 성장  (3-4 pages)
### 1.1 시장 규모 및 성장률
### 1.2 TAM / SAM / SOM 분석
### 1.3 Agentic AI 전환 트렌드
## 2. 경쟁 구도: KT 독점 체제의 균열        (4-5 pages)
### 2.1 한국 주요 플레이어 비교
### 2.2 글로벌 벤치마크
### 2.3 경쟁사별 약점 및 빈 공간
## 3. White Space: 성과 기반 과금 x 컴플라이언스 엔진  (3-4 pages)
### 3.1 2축 포지셔닝 맵
### 3.2 Perplexity 포지셔닝 비유
### 3.3 산업별 컴플라이언스 페인포인트
## 4. 전략적 시사점                       (2-3 pages)
### 4.1 핵심 데이터 포인트 요약 (색인)
### 4.2 후속 문서 연결점
## 출처                                  (1-2 pages)
```

### Data Point Requirements

- Minimum 10 unique data points with `MKT-DP-XX` IDs
- Each data point must have: (1) quantified fact, (2) named source, (3) confidence level
- Research.md already provides 18 verified data points (MKT-DP-01 through MKT-DP-18) -- use these as the primary pool
- Data points must appear both inline (footnote) AND in the Section 4.1 index table
