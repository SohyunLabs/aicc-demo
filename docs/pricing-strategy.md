# 뤼튼 AICC 가격 전략: 성과 기반 과금으로 구축형 시장을 전환하다


## Executive Summary

핵심 메시지는 세 가지로 요약된다.

- **시장 전환:** 글로벌 AICC 과금 모델이 좌석(seat) 기반에서 해결(resolution) 건당 과금으로 전환 중이다. Sierra AI(**$150M** ARR), Intercom(**$0.99/건**), Salesforce(**$0.10/액션**) 등이 선도하고 있으며, 한국 시장은 아직 구축형 + 좌석 기반이 지배적이다. 성과 기반 과금의 선점 기회가 열려 있다.
- **가격 전략:** 구축형의 높은 진입장벽(수억원 초기 투자)을 제거하고, "해결된 상담건당 과금"으로 고객사 리스크를 최소화한다. 3티어(Starter/Professional/Enterprise) 구조로 중소~대기업까지 전 세그먼트를 커버한다.
- **차별화:** 경쟁사(KT 구축형, Genesys 좌석 기반)가 "도입해야 비용 발생" 구조인 반면, 뤼튼은 "해결해야 비용 발생" 구조로 가치 증명을 과금 모델에 내장한다.


## 1. 구축형 가격 구조 (PRC-01)

한국 AICC 시장은 KT가 구축형 B2B 시장의 **69%**를 점유하는 독점 구조다(MKT-DP-04 참조). 구축형 시장의 비용 구조를 정확히 이해해야 SaaS 전환의 경제적 논거가 성립한다.


### 1.1 구축형 비용 구조

| 비용 항목 | 금액 범위 | 비고 |
|----------|----------|------|
| 초기 구축(SI) | 3-5억원 | PoC 6-12개월, 커스터마이징 포함 |
| 연 유지보수 | 5,000만원-1억원 | 초기 투자의 15-20%, 장애 대응+업그레이드 |
| 좌석 라이선스 | 월 ₩10-20만/석 | KT 기준 추정, 상담원 수 비례 |
| 하드웨어/인프라 | 5,000만원-1억원 | 서버, 네트워크, 보안 장비 |
| 커스터마이징 추가 개발 | 건당 3,000만-1억원 | 기능 변경 요청 시 별도 |

대형 금융지주의 경우 연간 AICC 관련 예산이 **50억원** 규모에 달하며(Phase 2 페르소나 참조), 이 중 상당 부분이 KT 구축형의 유지보수와 좌석 라이선스에 고정적으로 지출된다.


### 1.2 구축형의 구조적 한계

구축형 모델은 세 가지 구조적 한계를 안고 있다.

**첫째, 높은 진입장벽이다.** 중견 이하 기업은 수억원 초기 투자가 불가능하다. 보험 CXO(연 AICC 예산 10-30억원)에게도 3-5억원 PoC는 상당한 의사결정 부담이며, PoC 기간(6-12개월)까지 고려하면 도입 결정에서 실사용까지 1년 이상이 소요된다.

**둘째, 경직된 비용 구조다.** 상담 품질이 개선되든 안 되든 동일한 비용이 발생한다. 좌석 기반 과금은 상담원 수에 비례하므로, AI가 상담 효율을 높여 필요 상담원 수를 줄이더라도 즉각적인 비용 절감으로 연결되지 않는다. 성과와 비용의 연동이 없어 ROI 측정이 불가능하다.

**셋째, 전환 비용(Switching Cost)이 높다.** 한번 구축하면 2-3년 이상 잠금(lock-in)된다. 더 나은 솔루션이 나와도 전환 비용이 재구축 비용에 근접하기 때문에, 기존 솔루션에 만족하지 못하면서도 전환하지 못하는 상황이 발생한다.

그러나 구축형의 장점도 솔직히 인정해야 한다. 구축형은 **보안 통제권**(데이터가 고객사 인프라에 존재), **높은 커스터마이징 자유도**, 그리고 **대기업의 기존 조달 프로세스 적합성**이라는 명확한 강점을 갖는다. 대기업에게 구축형은 여전히 유효한 선택지이나, 중견/중소 시장에서는 비용 구조의 비합리성이 명확하다. **뤼튼의 SaaS 성과 기반 과금이 직접 경쟁하는 시장은 "구축형이 도달하지 못하는" 중견/중소 시장이며, 대기업 시장에서는 구축형 보완재로 포지셔닝한다.**


## 2. SaaS 성과 기반 과금 (PRC-02)

뤼튼 AICC의 SaaS 과금 모델은 "해결된 상담건당 과금(outcome-based pricing)"을 핵심으로 한다. AI가 해결하지 못한 상담에는 비용이 발생하지 않으므로, 고객사의 도입 리스크를 원천 제거한다.


### 2.1 "해결(Resolution)" 정의

성과 기반 과금의 핵심은 "해결"의 정의다. 해결의 기준이 불명확하면 과금 모델 전체의 신뢰가 훼손된다. 뤼튼은 해결을 다음 3가지 유형으로 다층 정의한다.

| 해결 유형 | 정의 | 과금 기준 | 측정 방법 |
|----------|------|----------|----------|
| **보이스봇 완결 해결** | AI가 상담원 개입 없이 고객 문의를 완전히 처리 | 건당 과금 (전액) | 고객이 추가 문의 없이 통화 종료 + 24시간 내 동일 건 재문의 없음 |
| **상담원 보조 해결** | Agent Assist 지원으로 상담원이 처리 | AHT 단축분 기반 과금 (할인 단가) | AI 미사용 대비 AHT 단축률 x 건수 |
| **미해결 (에스컬레이션)** | AI가 처리하지 못해 상담원에게 완전 이관 | 과금 없음 | Sierra AI와 동일 원칙 -- 해결하지 못하면 청구하지 않음 |

이 다층 정의의 핵심 원칙은 명확하다. "**해결하지 못하면 청구하지 않는다.**" 이는 Sierra AI가 실증한 성과 기반 과금의 핵심 원리이며, 뤼튼이 이를 한국 시장에 동일하게 적용한다.

> **[PRC-BP-02]** Sierra AI 성과 기반 과금 원칙: "미해결 건은 과금하지 않는다"
> -- 출처: Sierra AI Official Blog, Outcome-Based Pricing (MEDIUM)

> **[PRC-BP-03]** Salesforce Agentforce: **$2/대화** 모델에서 **$0.10/액션** 모델로 전환 (2025년)
> -- 출처: Salesforce Agentforce Pricing (HIGH)
> "대화 단위 과금은 '해결인지 아닌지' 판단 기준 문제가 발생. 액션 단위로 전환하여 측정 투명성 확보"

Salesforce의 사례는 "해결"의 정의가 왜 중요한지를 역설적으로 보여준다. $2/대화 모델이 실패한 이유는 "대화 1건이 해결인지 아닌지" 판단하기 어려웠기 때문이다. 뤼튼은 이 교훈을 반영하여, 보이스봇 완결 해결의 측정 기준을 "24시간 내 동일 건 재문의 없음"이라는 객관적 지표로 설정한다.

### 2.3 TCO 비교 시나리오

3개 산업별 가상 고객사를 설정하여, 구축형과 뤼튼 SaaS의 총소유비용(TCO)을 비교한다. 각 시나리오는 Phase 2에서 정의한 페르소나의 예산 규모와 상담 볼륨을 기반으로 설정했다.

**시나리오 A: 중견 금융사 (상담원 100명, 월 상담 5만건)**

| 항목 | 구축형 (KT 등) | 뤼튼 SaaS Professional |
|------|---------------|----------------------|
| 초기 투자 | 3-5억원 | 0원 |
| 연 유지보수 | 8,000만원 | 해결건당 과금에 포함 |
| 월 좌석 라이선스 | 100석 x ₩15만 = ₩1,500만 | - |
| 월 보이스봇 해결건당 | - | 1.5만건 x ₩2,000 = ₩3,000만 |
| 월 Agent Assist 건당 | - | 2만건 x ₩600 = ₩1,200만 |
| 미해결 건 | 비용 동일 | ₩0 (1.5만건) |
| **월 비용** | **₩1,500만 + 유지보수 분배** | **₩4,200만** |
| **1년 TCO** | **약 6-8억원** (초기 포함) | **약 5억원** |
| **3년 TCO** | **약 10-12억원** | **약 15억원** |
| **Break-even** | - | **약 18개월** |

- **TCO 논거:** 1년 차에는 SaaS가 유리하다(초기 투자 없음). 3년 이상이면 구축형이 유리해질 수 있으나, 자동 해결률이 향상될수록 SaaS 비용은 감소한다(해결 건수 증가 -> 미해결 건수 감소 -> 건당 단가 볼륨 효과). 금융사 CXO에게는 "초기 투자 없이 AI 상담 효과를 즉시 측정할 수 있다"는 점이 핵심 셀링 포인트다.

**시나리오 B: 중견 보험사 (상담원 200명, 월 상담 10만건)**

| 항목 | 구축형 (KT 등) | 뤼튼 SaaS Professional |
|------|---------------|----------------------|
| 초기 투자 | 5-8억원 | 0원 |
| 연 유지보수 | 1억원 | 해결건당 과금에 포함 |
| 월 좌석 라이선스 | 200석 x ₩15만 = ₩3,000만 | - |
| 월 보이스봇 해결건당 | - | 3만건 x ₩2,000 = ₩6,000만 |
| 월 Agent Assist 건당 | - | 4만건 x ₩600 = ₩2,400만 |
| **월 비용** | **₩3,000만 + 유지보수 분배** | **₩8,400만** |
| **1년 TCO** | **약 10-14억원** (초기 포함) | **약 10억원** |
| **3년 TCO** | **약 16-20억원** | **약 30억원** |

- **TCO 논거:** 보험사의 핵심 가치는 TCO가 아니라 **컴플라이언스**다. 설명의무 위반 1건의 법적 비용(과징금 수억원)을 감안하면, 전수 검사 기능의 리스크 감소 가치가 비용 차이를 상회한다. SaaS의 추가 가치는 즉시 도입(PoC 없음)과 컴플라이언스 엔진 포함이다. 보험 CXO에게는 "금감원 지적 사항 제로"가 비용보다 중요한 KPI다.

**시나리오 C: 이커머스 플랫폼 (상담원 50명, 월 상담 3만건, 피크 시즌 3-5배)**

| 항목 | 구축형 (KT 등) | 뤼튼 SaaS Starter |
|------|---------------|-------------------|
| 초기 투자 | 2-3억원 | 0원 |
| 연 유지보수 | 5,000만원 | 해결건당 과금에 포함 |
| 평시 월 비용 | 50석 x ₩15만 = ₩750만 | 1만건 x ₩2,500 + 1만건 x ₩800 = ₩3,300만 |
| 피크 시즌 월 비용 | ₩750만 (동일, 추가 석 필요 시 추가 비용) | 건수 비례 자동 스케일링 |
| **1년 TCO** | **약 3.5-5억원** | **약 4억원** |

- **TCO 논거:** 이커머스의 핵심 가치는 **탄력적 확장**이다. 피크 시즌(세일 기간) 문의량 3-5배 증가 시, 구축형은 추가 좌석 확보가 불가능하거나 지연된다. SaaS는 건수 비례 자동 스케일링으로 피크 시즌에 즉시 대응한다. 셀프 온보딩(2주 이내)으로 빠른 도입도 이커머스 VP의 핵심 요구사항이다.


## 3. 경쟁사 가격 포지셔닝 (PRC-03)

뤼튼의 가격이 "왜 이 수준인지"를 경쟁 맥락에서 논증한다. 가격만 나열하는 것이 아니라, 과금 모델의 구조적 차이(좌석 기반 vs 해결건당)가 만드는 경제적 차이를 비교한다.


### 3.1 경쟁사 가격 비교표

| 경쟁사 | 과금 모델 | 가격 | 대상 규모 | 출처 | 신뢰도 |
|--------|----------|------|----------|------|--------|
| KT 구축형 | SI 프로젝트 | 초기 3-5억원 + 연 유지보수 | 대기업 | 산업 보도 | MEDIUM |

### 3.2 과금 모델별 구조 비교

좌석 기반과 해결건당 과금의 구조적 차이를 Genesys 환산 비교로 논증한다.

Genesys CX3($155/seat/month = 약 ₩20.9만/석)를 사용하는 상담원 1인이 월 평균 2,000건을 처리한다고 가정하면, 건당 비용은 약 **₩105/건**이다. 그러나 이 비용은 해결 여부와 무관하게 발생한다. 뤼튼 SaaS Professional의 보이스봇 완결 해결 ₩2,000/건은 해결된 건에만 발생하며, 보이스봇이 처리하는 건은 상담원 투입이 불필요하다.

자동 해결률 **50%** 달성 시, 상담원 50명 -> 25명 수준으로 인력 최적화가 가능하며, 이때의 혼합 비용(보이스봇 해결 + 상담원 처리)은 순수 좌석 기반 대비 **30-40%** 절감된다. 핵심은 "건당 단가"가 아니라 "해결 여부에 따른 비용 발생 구조"다.

좌석 기반 과금은 AI 도입이 고도화될수록 역설적 문제를 안게 된다. AI가 상담을 자동 처리하여 필요 상담원 수가 줄면, 좌석 수(= 매출)도 줄어든다. 이것이 Genesys와 Five9가 직면한 구조적 딜레마이며, 성과 기반 과금으로의 전환 압력을 만드는 근본 원인이다.
