# 뤼튼 AICC SaaS 제품 로드맵


## Executive Summary

핵심 메시지는 세 가지로 요약된다.

- **MVP 6-9개월:** 가상 팀 **10명**으로 MVP 기능 **18개**를 출시한다. Enhance 12개(S-M Effort)는 기존 자산을 활용하여 병렬 개발하고, Build 6개(L Effort)는 Day 1 신뢰성을 위해 크리티컬 패스로 관리한다. 성과 기반 과금 엔진과 컴플라이언스 엔진이 Day 1부터 포함된다.
- **3단계 로드맵:** MVP(6-9개월) > 확장(+6개월) > 고도화(+6-12개월). PRD 우선순위 매트릭스(PRD-06 참조)의 MVP/Phase 2/Future 분류와 1:1 매핑하여 기획서 전체의 일관성을 확보한다.
- **리스크 관리:** 기술/시장/조직 **8개** 리스크를 식별하고, 각각에 완화 전략과 트리거 조건을 정의한다. LLM 할루시네이션, KT 69% 독점 시장 관성, AI/ML 인력 확보가 최우선 관리 대상이다.

본 로드맵은 시장 분석(Phase 1), 제품 요구사항 정의서(Phase 2), 가격 전략 및 PRD 완결(Phase 3), UX 와이어프레임(Phase 4)을 기반으로 작성되었다. 4개 선행 문서에서 정의된 기능, KPI, 가격, 시장 데이터를 실행 가능한 출시 계획으로 전환한 것이 본 문서의 핵심이다.


## 1. 개발 팀 구성 및 리소스 가정

뤼튼 AICC SaaS는 신사업부 또는 스타트업 규모의 가상 팀을 전제로 한다. 총 **10명** 규모로, Build L 6개 기능의 병렬 개발이 가능한 최소 구성이며 필요 시 **±2명** 조정을 가정한다.

| 역할 | 인원 | 주요 책임 | Phase별 배치 |
|------|------|----------|-------------|
| 백엔드 엔지니어 | 3명 | 보이스봇 코어 엔진, RAG 엔진 SaaS화, API 설계 | MVP 전체 |
| AI/ML 엔지니어 | 2명 | 할루시네이션 가드, 컴플라이언스 엔진, 감정 분석 | MVP + Phase 2 |
| 프론트엔드 엔지니어 | 2명 | 관리 콘솔 리뉴얼, Agent Assist UI, 셀프 온보딩 | MVP 전체 |
| QA 엔지니어 | 1명 | 전수 테스트, 컴플라이언스 검증, 성능 테스트 | MVP 전체 |
| PM | 1명 | 일정 관리, 이해관계자 조율, 리스크 모니터링 | 전체 |
| 디자이너 | 1명 | UI/UX 상세 설계, 디자인 시스템 구축 | MVP 전반부 |

총 **10명** 구성이다. AI/ML 엔지니어 2명은 MVP에서 할루시네이션 가드와 컴플라이언스 엔진(Build L)을 담당하고, MVP 출시 후 Phase 2에서 감정 분석과 품질 스코어링으로 전환한다. 디자이너는 MVP 전반부(M1-M4)에 상세 설계를 완료하고 이후 필요 시 파트타임으로 전환한다.

가상 팀 기준 대략적 추정. 실제 일정은 팀 역량/인프라 상태에 따라 조정 필요하며, 기존 구축 납품 경험이 있는 엔지니어 비율에 따라 Enhance 기능의 개발 속도가 크게 달라질 수 있다.


### 1.1 T-shirt 사이징 기준

기획서 단계에서 주/일 단위의 정밀 추정은 과도하므로, PRD-05에서 정의한 T-shirt 사이징을 가상 팀 기준 주수로 전환한다(PRD-05 참조).

| Size | 주수 | 해당 기능 유형 |
|------|------|-------------|
| S | 1-2주 | Enhance 기존 자산 래핑/설정 변경 수준 |
| M | 3-4주 | Enhance 기존 모듈 확장 + 일부 신규 로직 |
| L | 6-8주 | Build 신규 엔진/시스템 설계, 아키텍처 결정 필요 |
| XL | 10-12주 | Build 핵심 인프라 수준 대규모 개발 |

Enhance 14개 기능의 평균 Effort는 S-M(1-4주)이고, Build 6개 기능은 모두 L(6-8주)이다. **기존 자산 활용 비율 70%(14/20)**가 MVP 빠른 출시의 핵심 레버다(PRD-05 참조).


## 2. Phase별 기능 배포 계획

PRD-06 우선순위 매트릭스의 MVP/Phase 2/Future 분류를 3단계 로드맵에 1:1 매핑한다(PRD-06 참조). 배치 원칙은 "Day 1 신뢰성" 기준이며, 코어 밸류("틀리면 안 되는 상담")에 직결되는 기능은 개발 노력(Effort)과 무관하게 MVP에 포함한다.


### 2.1 MVP (Phase 1): Day 1 신뢰성 확보

MVP는 PRD-06의 **18개** 기능에 성과 기반 과금의 핵심인 해결 측정 엔진을 포함한다. Enhance 13개(S-M Effort)는 기존 5대 자산을 SaaS 환경에 맞게 전환/확장하여 제품 골격을 조기에 확보하고, Build 6개(L Effort)는 Day 1 신뢰성과 성과 과금에 필수인 핵심 엔진을 신규 개발한다.

| 기능 | 분류 | Effort | 주수(가상팀 기준) | 근거 |
|------|------|--------|-----------------|------|
| STT/TTS 엔진 | Enhance | S | 1-2주 | 기존 보이스봇 코어 엔진 SaaS 래핑. 모든 보이스 기능의 기초 (PRD-06 참조) |
| 대화 관리 | Enhance | S | 1-2주 | 기존 멀티턴 대화 엔진 SaaS 래핑. 자연스러운 상담 경험의 전제 조건 |
| Hybrid RAG 응답 | Enhance | M | 3-4주 | 기존 Hybrid RAG에 멀티테넌시 지식 격리 추가. "출처 기반 신뢰성" 코어 밸류의 기술적 실체 |
| 상담원 전환 | Enhance | S | 1-2주 | 기존 전환 로직에 Agent Assist 맥락 전달 추가. AI 한계 인식 시 안전망 |
| 실시간 전사 | Enhance | S | 1-2주 | 기존 STT에 화자 분리 적용. 통화 중 모든 AI 기능의 전제 조건 |
| 추천 응답 | Enhance | S | 1-2주 | 기존 RAG 응답을 상담원 화면에 표시. 신규 상담원도 베테랑 수준 안내 가능 |
| 약관 기반 정확 응답 | Enhance | M | 3-4주 | 보험 약관 특화 지식 그래프 확장. 보험 센터장 "교육 없이도 정확한 안내" 충족 |
| 상품별 정책 정확 안내 | Enhance | M | 3-4주 | 셀러별 정책 DB 연동 확장. 오안내 비용 연쇄(재문의+클레임+보상) 차단 |
| 주문/배송 상태 조회 | Enhance | M | 3-4주 | 물류 시스템 API 연동 확장. 단순 문의 자동 해결률 극대화로 ROI 조기 입증 |
| 고객 정보 자동 로드 | Enhance | M | 3-4주 | CRM 연동 자동 로드 모듈 확장. Agent Assist 전체 워크플로우의 시작점 |
| 실시간 요약 | Enhance | M | 3-4주 | 대화 요약 모듈 신규 확장. 긴 통화에서 핵심 포인트 한눈 파악 |
| 요약 자동 저장 | Enhance | M | 3-4주 | CRM 자동 기록 연동 확장. ACW 시간 50%+ 단축으로 운영 효율 즉시 체감 |
| 후처리 자동화 | Enhance | M | 3-4주 | 에스컬레이션/콜백 태깅 자동화 확장. 후속 조치 누락 방지 |
| 할루시네이션 가드 | Build | L | 6-8주 | 다층 검증 방어(출처 매칭+신뢰도 스코어+규정 위반 탐지) 신규 설계. Day 1 신뢰성 필수 (PRD-05 참조) |
| 상품 안내 컴플라이언스 | Build | L | 6-8주 | 금융위 AI 가이드라인 7대원칙 규정 엔진 신규 개발. Day 1 신뢰성 필수 |
| 불완전판매 방지 체크 | Build | L | 6-8주 | 금융소비자보호법 적합성 원칙 검증 로직 신규 개발. Day 1 신뢰성 필수 |
| 설명의무 이행 추적 | Build | L | 6-8주 | 보험업법 설명의무 체크리스트 자동 추적 엔진 신규 개발. Day 1 신뢰성 필수 |
| 컴플라이언스 경고 | Build | L | 6-8주 | 실시간 규정 위반 감지+알림 엔진 신규 개발. 전수 검사 전환의 핵심 |
| 해결 측정 엔진 | Build | L | 6-8주 | 성과 기반 과금의 핵심. 보이스봇 완결 해결/상담원 보조 해결/미해결 판정 엔진 신규 개발 (PRC-02 참조) |

PRD-06 기준 Enhance:Build 비율은 **13:5**이며, 해결 측정 엔진(Build L)을 포함하면 MVP 총 Build는 **6개**다. Enhance 13개는 S-M Effort(1-4주)로 병렬 개발 시 **1-2개월** 내 제품 골격 완성이 가능하다. Build 6개(모두 L Effort)는 MVP 출시 일정의 크리티컬 패스가 된다.

Build L 6개 중 직렬 의존성이 있는 크리티컬 패스는 **할루시네이션 가드 > 컴플라이언스 엔진(상품 안내/불완전판매/설명의무/경고) > 해결 측정 엔진(성과 과금)**이다. 할루시네이션 가드가 출처 매칭 검증의 기반을 제공하고, 이 위에 산업별 컴플라이언스 규칙이 적용되며, 최종적으로 "해결 여부"를 판정하는 과금 엔진이 완성되는 순서다. 이 직렬 경로가 약 **5-6개월**이며, 통합 테스트/QA 버퍼를 포함하면 MVP 전체 일정은 **6-9개월**이 된다.

**성과 기반 과금 엔진과 컴플라이언스 엔진이 MVP Phase 1에 포함된다.** 이것은 Day 1 신뢰성 기준의 직접적 결과이자, "성과 기반 과금 x 산업 특화 컴플라이언스"라는 핵심 차별화 전략의 실행 계획이다(PRC-02 참조).


### 2.2 확장 (Phase 2): 차별화 강화

MVP 출시 후 **+6개월** 범위다. PRD-06에서 Phase 2로 분류된 **2개** 기능을 개발하여 차별화를 강화한다(PRD-06 참조). 두 기능 모두 Medium-High Impact이나, 코어 밸류("틀리면 안 되는 응답") 외곽에 위치하여 Day 1 필수에서 제외되었다.

| 기능 | 분류 | Effort | 주수(가상팀 기준) | 근거 |
|------|------|--------|-----------------|------|
| 감정 분석 | Build | L | 6-8주 | NLP 감정 인식 + 수퍼바이저 알림 신규 개발. 코어 밸류 외곽, Day 1 신뢰성 직결 아님 |
| 품질 스코어링 | Build | L | 6-8주 | 전수 자동 QA + 코칭 포인트 도출 신규 개발. Medium-High Impact이나 코어 밸류 외곽 |

AI/ML 엔지니어 2명이 MVP 완료 후 감정 분석과 품질 스코어링으로 전환한다. 감정 분석은 고객 감정 실시간 모니터링으로 위험 고객을 조기 감지하고, 품질 스코어링은 기존 수동 샘플링(전체의 3-5%)을 전수 자동 QA로 전환하여 보험 페르소나의 핵심 페인포인트를 해결한다.


### 2.3 고도화 (Phase 3): 시장 확장

확장 Phase 완료 후 **+6-12개월** 범위다. 현 PRD 범위 외(Out of Scope)이므로 구체적 기능 정의 없이 방향성만 제시한다.

- 아웃바운드 보이스봇 -- 인바운드 자동 응대에서 아웃바운드 세일즈/알림 영역으로 확장 (v2 범위 -- 현 로드맵에서는 비전 수준으로만 배치)
- 다국어 지원 -- 영어/일본어 등 한국어 외 언어로 STT/TTS 및 RAG 엔진 확장 (현 PRD 범위 외 -- 방향성만 제시)
- 텍스트 채팅봇 채널 확장 -- 보이스 채널 외 카카오톡, 웹채팅 등 텍스트 채널로 확장 (현 PRD 범위 외 -- 방향성만 제시)

고도화 Phase의 구체적 범위와 우선순위는 MVP 출시 후 시장 반응과 고객 피드백에 기반하여 결정한다.

**전체 기능 배포 계획이 PRD-06 우선순위 매트릭스와 1:1 매핑되어, 시장 분석 > PRD > 가격 전략 > UX > 로드맵으로 이어지는 기획서 전체의 일관성을 확보했다.**

### 4.1 마일스톤-KPI 매핑

각 마일스톤에 PRD-07의 KPI 목표치를 매핑하여, 제품 성장 단계별 성공 기준을 정의한다.

| 마일스톤 | 시점 | 핵심 KPI 목표 | 출처 |
|---------|------|-------------|------|
| 알파 테스트 | M3-M4 | 내부 기능 검증 완료, 핵심 시나리오(보이스봇 자동 응대 + 상담원 전환) 동작 확인 | 내부 |
| 베타 테스트 | M6-M7 | 파일럿 고객 1-2사 투입, 실환경 검증, 컴플라이언스 엔진 금융/보험 시나리오 통과 | 내부 |
| MVP 출시 | M8-M9 | 자동 해결률 **35%**, 할루시네이션 발생률 **<3%**, CSAT **78점** | PRD-07 90일 목표 |
| 안정화 | M12 | 자동 해결률 **50%**, AHT 단축 **25%**, CSAT **83점** | PRD-07 180일 목표 |
| 확장 완료 | M15-M18 | 자동 해결률 **65%**, 할루시네이션 발생률 **<0.5%**, 컴플라이언스 위반 사전 차단 **95%+** | PRD-07 1년 목표 |

MVP 출시(M8-M9) 시점의 KPI 목표는 PRD-07 90일 목표와 일치한다. 이는 MVP 출시 후 90일 시점에서 자동 해결률 **35%**와 할루시네이션 발생률 **<3%**를 달성해야 함을 의미하며, 이 수치가 성과 기반 과금 모델의 실행 가능성을 검증하는 첫 번째 관문이다. 안정화(M12)와 확장 완료(M15-M18)의 KPI는 각각 180일/1년 목표로, 제품이 성숙함에 따라 자동 해결률과 컴플라이언스 차단율이 점진적으로 상향되는 성장 궤적을 보여준다(PRD-07 참조).
